<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (owner) → /customers/[id] RSC | `id` is attacker-controllable in the URL — could be another business's customer |
| Browser → addAppointmentAction / deleteAppointmentAction (POST) | Server Actions are POST-reachable; formData (customerId, appointment id, date, service) is untrusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `/customers/[id]` leaking another business's customer + visits | mitigate | `findFirst({ where: { id, businessId: business.id } })` → `notFound()`; never `findUnique` by id alone (SC#4, Pitfall 5) |
| T-02-02 | Tampering | `deleteAppointmentAction` deleting another business's appointment via crafted id | mitigate | `deleteMany({ where: { id, businessId: business.id } })` re-checks ownership (D-12); a foreign id matches 0 rows |
| T-02-03 | Elevation of Privilege | `addAppointmentAction` writing a visit onto another business's customer | mitigate | verify `customer.findFirst({ where: { id: customerId, businessId } })` BEFORE insert; create with `businessId: business.id` |
| T-02-04 | Spoofing | Unauthenticated POST to either action | mitigate | `requireCurrentBusiness()` (→ `requireCurrentUser`) enforced INSIDE each action, not just UI |
| T-02-05 | Tampering | Duplicate visits inflating Phase-2 cadence via repeated submits | mitigate | D-13 dedup (same customer+date+service rejected); midnight-normalized date for byte-clean equality |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No packages installed — RESEARCH §Package Legitimacy Audit: N/A |
</threat_model>
