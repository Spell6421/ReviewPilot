<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → importAppointmentsAction (POST + file upload) | Server Action is POST-reachable; the uploaded CSV (names, phones, dates, services) is fully untrusted |
| Client preview (parseAppointmentsCsv in browser) → server re-parse | the client dry-run is advisory only; the server re-parse + validation is authoritative |
| CSV phone → existing-customer match / auto-create | a mismatched normalization could attach a visit to the wrong customer or duplicate one |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | Unauthenticated POST to importAppointmentsAction | mitigate | `requireCurrentBusiness()` enforced inside the action (auth in the action, not UI) |
| T-04-02 | Tampering | Untrusted CSV rows (bad phone/date, injection in service text) inserted blindly | mitigate | authoritative SERVER re-parse via `parseAppointmentsCsv`; only zero-error rows inserted; bad rows surfaced (SC#2); service stored as plain text, rendered via React (auto-escaped) |
| T-04-03 | Elevation of Privilege | Phone match / auto-create / insert hitting another business | mitigate | every `customer.findFirst` / `customer.create` / `appointment.create` scoped by `businessId: business.id`; auto-created customer owned by the current business |
| T-04-04 | Tampering | Wrong-customer attach or duplicate customers from phone-format divergence | mitigate | match on the parser's verbatim `normalizePhone` E.164 (exact string, same as inbound-reply) — Pitfall 3 |
| T-04-05 | Tampering | Re-import inflating visit counts → corrupts Phase-2 cadence | mitigate | D-10/D-13 dedup (same customer+date+service skipped + reported); midnight-normalized date for byte-clean equality (Pitfall 4) |
| T-04-06 | Tampering | Cache drift if recompute skipped on the bulk path | mitigate | one `recomputeLastAppointment` per distinct affected customer; invariant SQL = 0 gate (Pitfall 1) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No packages installed — RESEARCH §Package Legitimacy Audit: N/A (papaparse already pinned) |
</threat_model>
